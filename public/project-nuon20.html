Een nieuwe website voor Nuon

Good designers redesign, great designers realign.
Cameron Moll

In 2013 heb ik voor Nuon de frontend voor het vernieuwde Nuon.nl verzorgd. Naast een nieuw CMS, een vernieuwde backend voor verschillende tools en een contentmigratie, was er een grote update van de frontend nodig. Het design was verouderd en ook de CSS kon door een aantal jaren legacy een opfrisbeurt gebruiken.

Op basis van een groteendeels bestaande markup, is een nieuwe ontwerp geïmplementeerd. Hierbij was een van de uitgangspunten dat het ontwerp zo dicht mogelijk bij het oude ontwerp moest blijven. Het nieuwe ontwerp moest voornamelijk zorgen voor een moderner uiterlijk en meer consistentie.

In het oude ontwerp werd veel gebruik gemaakt van afgeronde hoeken en schaduwen. Deze zijn voor een groot deel weggelaten in het nieuwe ontwerp, waardoor de site niet alleen moderner aanvoelt, maar ook sneller geworden is.

Beter voorbereid op de toekomst

De CSS is volledig opnieuw geschreven, gebaseerd op OOCSS (object georienteerd CSS), wat zorgt voor herbruikbare code. Dit heeft geresulteerd in een CSS die aanzienlijk (~50%) kleiner is geworden. Daarnaast is gebruik gemaakt van CSS in plaats van Javascript waar mogelijk wat voor een verdere optimalisatie in de laadtijd gezorgd heeft.

Het project heeft door deze en andere wijzigingen voor een verbetering gezorgd voor de laadtijd van de site, de toegankelijkheid en de algehele gebruikerservaring. Dit heeft ook een positieve invloed gehad op de SEO (zoekmachine optimalisatie) en de conversie van de site.

De OOCSS opzet zorgt dat de site beter voorbereid is op de toekomst, het is makkelijker geworden om wijzigingen op het design door te voeren en door goed hergebruik van styles blijft de consistentie behouden.

Pattern Library

Als onderdeel van het project heb ik een Pattern Library opgeleverd waarin alle componenten die op de site gebruikt worden gedocumenteerd staan met de bijbehorende HTML.

Deze library wordt intern gebruikt als documentatie voor developers en contentmanagers, maar ook extern door leveranciers als naslagwerk bij het bouwen van tools.

Behalve als naslagwerk, zijn Pattern Libraries ook nuttig tijdens een project, met name bij een Atomic Design aanpak. Bij deze aanpak wordt een website gezien als een verzameling componenten, die op deze manier apart ontworpen kunnen worden.
